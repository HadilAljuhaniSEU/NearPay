## Fields added in customer journey phase
```ts
merchantName?: string;       // business name, stored at create time from merchant.name
referenceNumber?: string;    // NP-YYYY-XXXXXX format, generated by generateReferenceNumber() in lib/tokens.ts
paymentStatus?: 'unpaid' | 'partial' | 'paid';  // updated by applyPaymentToDebt
approvedAt?: Timestamp;      // set by updateDebtApproval when approvalStatus='approved'
paidAt?: Timestamp;          // set by applyPaymentToDebt when remainingAmount reaches 0
disputeReason?: string;      // set by updateDebtDispute
disputedAt?: Timestamp;      // set by updateDebtDispute
```

## createDebt return type
`Promise<{ id: string; approvalToken: string; paymentToken: string; referenceNumber: string }>`

## Key service functions
- `updateDebtApproval(id, status)` — also sets `approvedAt` when status='approved'
- `updateDebtDispute(id, reason)` — sets approvalStatus='disputed', disputeReason, disputedAt
- `applyPaymentToDebt(id, paid, remaining)` — sets paymentStatus and paidAt
- `subscribeDebtsByCustomerPhone(phone, callback)` — real-time query by customerPhone

**Why:** All fields are optional to maintain backward compatibility with existing debt documents that were created before this phase.
